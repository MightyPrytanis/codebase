# Beta Test Feedback Process

**Date:** 2025-12-29  
**Audience:** Beta Testers  
**Purpose:** Guide for submitting feedback during beta testing

---

## Overview

Your feedback is essential to improving Cyrano, LexFiat, and Arkiver. This guide explains how to submit feedback effectively.

---

## Feedback Types

### 1. Bug Reports

**When to Submit:**
- Application crashes or errors
- Features don't work as expected
- Unexpected behavior
- Performance issues

**What to Include:**
- **Title:** Brief description (e.g., "Dashboard doesn't load on mobile")
- **Description:** What happened
- **Steps to Reproduce:** Step-by-step how to recreate the issue
- **Expected Behavior:** What should have happened
- **Actual Behavior:** What actually happened
- **Screenshots:** If applicable
- **System Info:** Platform, browser (if web), version

**Example:**
```
Title: Dashboard doesn't load on mobile Safari

Description: When accessing LexFiat dashboard on iPhone Safari, 
the page loads but shows blank white screen.

Steps to Reproduce:
1. Open Safari on iPhone
2. Navigate to cognisint.com/beta
3. Login
4. Click "LexFiat Dashboard"
5. Page shows blank white screen

Expected: Dashboard should load and display widgets

Actual: Blank white screen, no content visible

Screenshots: [attached]
System: iPhone 14, iOS 17.2, Safari
```

### 2. Feature Requests

**When to Submit:**
- You have an idea for improvement
- Missing functionality you need
- Enhancement to existing feature

**What to Include:**
- **Title:** Feature name
- **Description:** What the feature should do
- **Use Case:** Why it's needed
- **Priority:** How important (High/Medium/Low)

**Example:**
```
Title: Add keyboard shortcuts for common actions

Description: Add keyboard shortcuts for frequently used actions 
like creating new matter, saving document, etc.

Use Case: Power users could work faster with keyboard shortcuts 
instead of clicking through menus.

Priority: Medium
```

### 3. General Feedback

**When to Submit:**
- Overall impressions
- Usability observations
- Workflow suggestions
- General comments

**What to Include:**
- **Title:** Topic
- **Description:** Your feedback
- **Context:** Where/when you noticed this

### 4. Praise/Positive Feedback

**When to Submit:**
- Features you love
- Things that work well
- Positive experiences

**Why It Matters:**
- Helps us know what to keep
- Motivates the team
- Guides future development

## How to Submit Feedback

### Method 1: Via Portal (Recommended)

1. **Log into Beta Portal**
2. **Click "Feedback" in dashboard**
3. **Select feedback type**
4. **Fill out form**
5. **Attach files if needed** (screenshots, logs)
6. **Click "Submit"**
7. **Receive confirmation email**

**Time:** < 3 minutes

### Method 2: Via Cyrano Pathfinder

1. **Click "Get Help" on any page**
2. **Say:** "I want to submit feedback"
3. **Pathfinder guides you through:**
   - Selecting feedback type
   - Collecting information
   - Submitting feedback
4. **Receive confirmation**

**Time:** < 5 minutes (with guidance)

### Method 3: Via Email

**Email:** beta-feedback@cognisint.com

**Include:**
- Subject: Feedback Type - Brief Description
- Your beta tester email
- Detailed description
- Attachments if applicable

## Feedback Best Practices

### Be Specific
- ✅ "Dashboard loads slowly on mobile" (specific)
- ❌ "App is slow" (too vague)

### Include Context
- When did it happen?
- What were you trying to do?
- What page/feature were you using?

### Provide Steps
- Step-by-step reproduction helps us fix issues faster
- Even if it seems obvious, include all steps

### Attach Evidence
- Screenshots of errors
- Screen recordings of issues
- Log files (if requested)
- Error messages (copy exact text)

### Be Constructive
- Focus on the issue, not frustration
- Suggest improvements when possible
- Explain impact on your workflow

## Tips for Effective Feedback

1. **Submit Early**
   - Don't wait to collect multiple issues
   - Submit as you find them
   - Fresh in your mind

2. **Be Detailed**
   - More information = faster resolution
   - Include all relevant context
   - Don't assume we know your setup

3. **Test Before Submitting**
   - Try to reproduce the issue
   - Check if it's consistent
   - Note any patterns

4. **Follow Up**
   - Check status of your feedback
   - Respond to questions
   - Verify fixes when notified
